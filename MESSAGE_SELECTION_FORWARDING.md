# Реализация выделения и пересылки сообщений

## Описание

Реализована полная функциональность выделения множественных сообщений и их пересылки в другие чаты, аналогично современным мессенджерам (Telegram, WhatsApp).

## Функциональность

### 1. Режим выделения

**Вход в режим:**
- **Долгое нажатие** на сообщение (500мс) на мобильных устройствах
- **Пункт "Выбрать"** в контекстном меню (правый клик / долгое нажатие)

**Возможности:**
- Клик по сообщению или чекбоксу для выделения/снятия выделения
- Визуальная подсветка выделенных сообщений
- Счетчик выделенных сообщений
- Автоматический выход при снятии всех выделений
- Выход по кнопке "Отмена" или клавише ESC

### 2. Панель действий

При активном режиме выделения внизу экрана появляется панель с:
- **Счетчиком** выделенных сообщений
- **Кнопкой "Переслать"** - открывает диалог выбора чата
- **Кнопкой "Удалить"** - удаляет выделенные сообщения (только свои)
- **Кнопкой "Отмена"** - выход из режима

### 3. Диалог пересылки

**Компоненты:**
- Заголовок с количеством сообщений
- Поисковая строка для фильтрации чатов
- Список доступных чатов (исключая текущий)
- Аватары и названия чатов
- Превью последнего сообщения

**Функции:**
- Поиск чата по названию
- Клик по чату для пересылки
- Автоматическая загрузка списка чатов
- Обработка ошибок

### 4. Backend API

**Endpoints:**

#### `POST /api/v1/communications/messages/forward/`
Пересылка сообщений в другой чат.

**Request:**
```json
{
  "message_ids": [1, 2, 3],
  "target_chat_id": 5
}
```

**Response:**
```json
{
  "ok": true,
  "forwarded_count": 3,
  "message_ids": [10, 11, 12]
}
```

**Проверки:**
- Доступ к целевому чату
- Права на отправку сообщений в целевой чат
- Доступ к исходным сообщениям
- Копирование вложений

#### `POST /api/v1/communications/messages/bulk-delete/`
Массовое удаление сообщений.

**Request:**
```json
{
  "message_ids": [1, 2, 3]
}
```

**Response:**
```json
{
  "ok": true,
  "deleted_count": 3
}
```

**Ограничения:**
- Можно удалять только свои сообщения

### 5. Модель данных

**Новые поля в Message:**
```python
is_forwarded = BooleanField(default=False)
forwarded_from_message_id = IntegerField(null=True, blank=True)
forwarded_from_author = ForeignKey(User, null=True, blank=True)
```

**Сериализация:**
```javascript
{
  "id": 123,
  "content": "Текст сообщения",
  "is_forwarded": true,
  "forwarded_from": {
    "author_id": 5,
    "author_name": "Иван Иванов",
    "message_id": 100
  }
}
```

## Использование

### Как выделить сообщения

**На мобильных:**
1. Долго нажать на сообщение (500мс)
2. Появятся чекбоксы у всех сообщений
3. Тапать по сообщениям для выделения

**На десктопе:**
1. Правый клик на сообщении
2. Выбрать "Выбрать" в меню
3. Кликать по сообщениям для выделения

### Как переслать

1. Выделить сообщения
2. Нажать "Переслать" в панели внизу
3. Выбрать чат из списка
4. Сообщения автоматически отправятся

### Как удалить

1. Выделить свои сообщения
2. Нажать "Удалить" (кнопка появится только для своих сообщений)
3. Подтвердить удаление
4. Сообщения удалятся

## WebSocket события

При пересылке отправляется обычное событие `chat_message` с флагом `is_forwarded`:

```javascript
{
  "type": "chat_message",
  "payload": {
    "id": 123,
    "content": "...",
    "is_forwarded": true,
    "forwarded_from": {
      "author_name": "Иван Иванов",
      "message_id": 100
    }
  }
}
```

При удалении:
```javascript
{
  "type": "chat_message_deleted",
  "message_id": 123
}
```

## Безопасность

**Проверки доступа:**
1. Пользователь должен иметь доступ к исходным сообщениям
2. Пользователь должен иметь доступ к целевому чату
3. Пользователь должен иметь права на отправку в целевой чат
4. Можно удалять только свои сообщения

**CSRF защита:**
- Все POST запросы требуют CSRF токен
- Декоратор `@csrf_protect` на всех endpoints

**Авторизация:**
- Декоратор `@login_required` на всех endpoints
- Проверка через `user_can_access_chat()`

## Тестирование

**Сценарии для тестирования:**

1. **Выделение**
   - Долгое нажатие на сообщение
   - Выделение нескольких сообщений
   - Снятие выделения
   - Выход по ESC

2. **Пересылка**
   - Пересылка 1 сообщения
   - Пересылка нескольких сообщений
   - Пересылка с вложениями
   - Пересылка в группу
   - Пересылка в личный чат

3. **Удаление**
   - Удаление своих сообщений
   - Попытка удалить чужие (кнопка не должна появиться)
   - Массовое удаление

4. **Поиск чатов**
   - Поиск по названию
   - Фильтрация результатов
   - Пустой результат

5. **Ошибки**
   - Нет доступа к чату
   - Нет прав на отправку
   - Сетевая ошибка

## Известные ограничения

1. Нельзя пересылать удаленные сообщения
2. Нельзя пересылать системные сообщения
3. Лимит на количество сообщений (определяется backend)
4. История пересылок не отслеживается (можно расширить)
