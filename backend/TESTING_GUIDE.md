## ✅ Что тестировать

### ЭТАП 1-3: Автоматические уведомления (Signal Integration)

**1. Чат - Communications** 💬
- Отправьте сообщение в чат → получатель получит уведомление
- Упомяните @email в сообщении → уведомление об упоминании
- Ответьте на сообщение → автор получит уведомление

**2. Заявки - Requests** 📋
- Создайте заявку → руководители получат уведомление
- Добавьте комментарий → автор получит уведомление
- Одобрите/отклоните заявку → автор получит уведомление

**3. Документы - Documents** 📄
- Загрузите документ → получатели получат уведомление
- Ознакомьтесь с документом → автор получит уведомление (если все ознакомились)

**4. Календарь - Calendar** 📅
- Создайте событие → участники получат уведомление
- Измените событие → участники получат уведомление
- Удалите событие → участники получат уведомление об отмене

**5. Лента - Feed** 📰
- Создайте пост → пользователи получат уведомление (по типу)
- Комментируйте пост → автор получит уведомление
- Поставьте лайк → автор получит уведомление

### ЭТАП 4: UI и UX

**1. Колокольчик в Navbar** 🔔
- Badge показывает количество непрочитанных
- Клик открывает dropdown с последними 5 уведомлениями
- Кнопка "Все" ведет на полный список
- Кнопка "Настройки" ведет на настройки

**2. WebSocket Real-time** ⚡
- Новое уведомление появляется мгновенно
- Badge обновляется без перезагрузки
- Список в dropdown обновляется

**3. Браузерные уведомления** 🖥️
- При первом посещении запрашивается разрешение
- Новые уведомления показываются в ОС
- Клик по уведомлению переходит на объект
- Уведомление отмечается прочитанным

**4. Звуковые уведомления** 🔊
- При новом уведомлении звучит короткий звук
- Переключатель в настройках: вкл/выкл
- Настройка сохраняется в браузере

**5. Список уведомлений** 📋
Откройте: `/notifications/`

- **Фильтр по категории:** выберите из 8 категорий
- **Фильтр по статусу:** все/прочитанные/непрочитанные
- **Поиск:** введите текст и нажмите "Найти"
- **Пагинация:** переключение между страницами
- **Действия:**
  - "Посмотреть" → переход к объекту
  - "Прочитано" → отметить прочитанным
  - Корзина → удалить уведомление
  - "Отметить все прочитанными" → массовое действие

**6. Настройки** ⚙️
Откройте: `/notifications/settings/`

- **Звук:** переключатель (глобально)
- **По категориям:** 8 карточек
  - Главный переключатель (вкл/выкл категорию)
  - Каналы: Веб (работает), Email/Telegram (скоро)
- **Автосохранение:** изменения сразу отправляются на сервер
- **Toast:** всплывает "Настройки сохранены"

## 🧪 Генератор тестовых уведомлений

Для быстрого тестирования используйте скрипт:

```bash
cd backend
python generate_notification.py
```

**Меню:**
1. Новое сообщение в чате
2. Новый документ
3. Одобрена заявка
4. Новое событие в календаре
5. Новая публикация
6. Системное уведомление
7. Создать 5 разных уведомлений

После создания:
- Проверьте колокольчик (должен обновиться badge)
- Должно прийти браузерное уведомление
- Должен воспроизвестись звук
- Откройте список - уведомление должно быть там

## 🔧 Отладка

### Консоль браузера
Откройте DevTools (F12) → Console

Смотрите логи:
```
[Notifications] WebSocket connected
[Notifications] Received: {type: 'new_notification', ...}
[Notifications] New notification: {...}
[Notifications] Browser notification shown
```

### Django logs
Смотрите терминал где запущен runserver

### WebSocket
Проверьте в DevTools → Network → WS:
- Соединение установлено
- Сообщения приходят
- Нет ошибок

### API
Проверьте запросы в DevTools → Network → Fetch/XHR:
- `/api/notifications/` - список
- `/api/notifications/count/` - счетчик
- `/api/notifications/{id}/read/` - прочитать
- `/api/notifications/settings/` - настройки
