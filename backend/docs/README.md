## 📝 Правила

### Добавление новой документации:

1. **Guides** - постоянные инструкции, reference материалы
2. **Troubleshooting** - временные документы, после решения → удалить
3. **Architecture** - долгосрочная документация, обновлять при изменениях
4. **Testing** - планы тестирования, после завершения → архивировать

```markdown
...
```

## 🔄 Жизненный цикл

1. **Создание** - документируйте новую фичу/проблему
2. **Актуализация** - обновляйте при изменениях
3. **Архивация** - после завершения/решения
4. **Удаление** - если больше не актуально
