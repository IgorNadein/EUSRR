# Тестирование исправлений дублирующей логики

**Дата:** 12.01.2026  
**Контекст:** Исправлены 2 критические проблемы дублирования логики

## Что было исправлено

### 1. ✅ MessageContextMenu - убрана двойная обработка
- **Было:** MutationObserver + Event listener = каждое сообщение обрабатывалось дважды
- **Стало:** Только Event listener на `chat:message-added`

### 2. ✅ Day Dividers - унифицирован метод создания
- **Было:** chatHistoryLoader использовал старый `createDayDivider` (минимальный стиль)
- **Стало:** Все используют messageRenderer (единый стиль с фоном)

### 3. ✅ Убраны неиспользуемые импорты
- userWebSocket.js больше не импортирует `createDayDivider`, `createMessageElement`

**Статус:** НЕ исправлено в этом коммите

**Описание:**
Страница загружается, но потом визуально "прыгает" к последнему сообщению.

**Тест:**
1. Открыть чат
2. ВНИМАТЕЛЬНО смотреть на момент загрузки
3. Заметить прыжок scroll'а

**Возможные причины:**
- CSS transitions вызывают reflow
- Browser paint происходит ПОСЛЕ установки scrollTop
- Нужен двойной requestAnimationFrame
- Что-то в base.html делает дополнительный scroll
