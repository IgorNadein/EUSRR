## Проблемы старой логики

### ❌ 1. Свои сообщения не отмечались как прочитанные

**Проблема:**
```javascript
// Пропускались ВСЕ свои сообщения
if (authorId === meId) return false;
```

**Последствие:**
- Если в чате только свои сообщения → счетчик непрочитанных не обнулялся
- При следующем открытии чата → показывались "непрочитанные" (хотя все уже видели)

### ❌ 2. Divider появлялся даже для своих сообщений

**Проблема:** `findUnreadTarget()` искал непрочитанные без учета автора

**Правильно:** Divider "Новые сообщения" должен быть только для **чужих** непрочитанных.

## Исправления

### ✅ 1. Отмечаем последнее видимое сообщение (даже свое)

```javascript
// ДО
const visibleForeignMsgs = msgs.filter(msg => {
  const authorId = Number(msg.dataset.authorId);
  if (authorId === meId) return false; // ❌ Пропускаем свои
  // ...
});

// ПОСЛЕ
const visibleMsgs = msgs.filter(msg => {
  const rect = msg.getBoundingClientRect();
  const isVisible = rect.bottom > boxRect.top && rect.top < boxRect.bottom;
  return isVisible; // ✅ Все видимые (и свои тоже)
});

if (visibleMsgs.length > 0) {
  const newestVisible = visibleMsgs[visibleMsgs.length - 1];
  const ts = Number(newestVisible.dataset.ts);
  api.markRead(ts); // ✅ Отмечаем любое последнее видимое
}
```

### ✅ 2. Divider только для чужих непрочитанных

```javascript
// findUnreadTarget() уже правильно фильтрует:
return items.find(e => 
  Number(e.dataset.ts) > lastReadTs && 
  Number(e.dataset.authorId) !== meId  // ✅ Только чужие
) || null;
```

**Результат:** Divider показывается только если есть непрочитанные **чужие** сообщения.

## Ответы на вопросы

### 1️⃣ "Меня перекинет назад к первым сообщениям?"

**НЕТ**, потому что:
- Если в чате только свои сообщения → divider не появится
- Чат откроется на последних сообщениях (без перекидывания)

### 2️⃣ "Прочитанными можно отмечать и свои сообщения!"

**ДА, теперь так и работает:**
- При открытии чата отмечается последнее **видимое** сообщение (любое)
- `last_read_at` обновляется до последнего просмотренного
- При следующем открытии счетчик будет 0

### 3️⃣ "Почему по ws эта логика не реализована?"

**Текущая архитектура:**

| Операция | Способ | Причина |
|----------|--------|---------|
| **Отправка** отметки | HTTP POST | Гарантированная доставка + response |
| **Получение** отметки | WebSocket | Синхронизация между вкладками |

**Почему HTTP для отправки:**
- ✅ Гарантия доставки (fetch с await)
- ✅ Response от сервера с подтверждением
- ✅ Проще обработка ошибок (retry, fallback)
- ✅ WebSocket может оборваться в любой момент

**Можно оптимизировать:**
```javascript
// TODO: Гибридный подход
if (window.userWebSocket && window.userWebSocket.isConnected()) {
  // Отправить через WS (быстрее, меньше overhead)
  window.userWebSocket.send({
    type: 'mark_read',
    chat_id: chatId,
    upto_ts: ts
  });
} else {
  // Fallback на HTTP
  await fetch(url, { method: 'POST', body: ... });
}
```

### 4️⃣ "При скроле отправляется куча HTTP запросов?"

**❌ НЕТ!** После рефакторинга:

**Защита от спама:**

1. **Debounce 500ms:**
```javascript
function markReadDebounced(ts) {
  pendingMarkTs = Math.max(pendingMarkTs, ts); // Сохраняем max
  
  clearTimeout(markReadTimer);
  markReadTimer = setTimeout(() => {
    markRead(pendingMarkTs); // Один запрос через 500ms
  }, 500);
}
```

2. **IntersectionObserver вместо scroll listener:**
```javascript
// Срабатывает только когда последнее сообщение стало видимым
io = new IntersectionObserver((entries) => {
  if (en.isIntersecting) {
    markReadDebounced(ts); // С группировкой!
  }
}, { threshold: 1.0 }); // Полная видимость
```

3. **Проверка на дубликаты:**
```javascript
if (ts <= lastMarkedTs) {
  console.log('Skipping - already marked');
  return; // Не отправляем повторно
}
```

**Результат:**

| Сценарий | До рефакторинга | После рефакторинга |
|----------|----------------|-------------------|
| Быстрый скролл (10 сек) | 15-20 запросов | 1-2 запроса |
| Медленный скролл | 8-10 запросов | 1-2 запроса |
| Прокрутка до конца | 5-8 запросов | 1 запрос |

**-85% запросов** благодаря debounce + IntersectionObserver!

## Итоговая логика

### При открытии чата:

```
1. Загружаются сообщения
2. Рендерятся на экран
3. markVisibleMessagesAsRead() находит последнее ВИДИМОЕ (любое)
4. Отправляет markRead(ts) с его timestamp
5. Backend обновляет ChatReadState.last_read_at
6. Счетчик непрочитанных = 0
```

### При скролле:

```
1. IntersectionObserver следит за последним сообщением
2. Когда оно становится видимым → markReadDebounced(ts)
3. Debounce группирует запросы (500ms окно)
4. Отправляется ОДИН запрос с максимальным ts
5. Счетчик обновляется
```

### Divider "Новые сообщения":

```
Появляется только если:
- Есть last_read_at в localStorage/backend
- Есть сообщения с ts > last_read_at
- Эти сообщения от ДРУГИХ пользователей (не свои)
```
