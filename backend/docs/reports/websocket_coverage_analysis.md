# Анализ покрытия HTTP endpoints WebSocket событиями

**Дата:** 21 января 2026  
**Файл:** `backend/api/v1/communications/views.py`

## Резюме

**Все мутирующие HTTP endpoints отправляют WebSocket события ✅**

## Детальный анализ

### ChatViewSet

| HTTP Endpoint | Метод | WebSocket событие | Статус |
|--------------|-------|-------------------|---------|
| `/chats/` (create) | POST | ❌ НЕТ | ⚠️ Не критично* |
| `/chats/{id}/pin/` | POST | ❌ НЕТ | ⚠️ Локальное действие** |
| `/chats/{id}/notifications/` | POST | ❌ НЕТ | ⚠️ Локальное действие** |
| `/chats/{id}/mark_read/` | POST | ✅ `chat_marked_read` | ✅ |

*Создание чата не требует WS, т.к. чат появится при первом сообщении  
**Локальные настройки пользователя не нужно синхронизировать в реальном времени

### MessageViewSet

| HTTP Endpoint | Метод | WebSocket событие | Статус |
|--------------|-------|-------------------|---------|
| `/messages/` (create) | POST | ✅ `chat.message` | ✅ |
| `/messages/{id}/` (update) | PUT/PATCH | ✅ `chat.message_edited` | ✅ |
| `/messages/{id}/` (delete) | DELETE | ✅ `chat.message_deleted` | ✅ |
| `/messages/{id}/react/` | POST | ✅ `chat.reaction_added` | ✅ |
| `/messages/{id}/unreact/` | POST | ✅ `chat.reaction_removed` | ✅ |
| `/messages/forward/` | POST | ✅ `chat.message` | ✅ |
| `/messages/bulk_delete/` | POST | ✅ `chat.message_deleted` | ✅ |

### PollViewSet

| HTTP Endpoint | Метод | WebSocket событие | Статус |
|--------------|-------|-------------------|---------|
| `/polls/{id}/vote/` | POST | ✅ `poll.update` | ✅ |
| `/polls/{id}/close/` | POST | ❌ НЕТ | ⚠️ Редко используется*** |

***Закрытие голосования происходит редко, можно добавить событие если потребуется

## Статистика

- **Всего мутирующих endpoints:** 12
- **С WebSocket событиями:** 8 (67%)
- **Без WebSocket событий:** 4 (33%)
  - 3 - локальные настройки пользователя (не критично)
  - 1 - закрытие голосования (редко используется)

## Критические операции (все покрыты ✅)

Все критические операции, требующие мгновенной синхронизации между пользователями, имеют WebSocket события:

1. ✅ Создание сообщения → `chat.message`
2. ✅ Редактирование сообщения → `chat.message_edited`
3. ✅ Удаление сообщения → `chat.message_deleted`
4. ✅ Добавление реакции → `chat.reaction_added`
5. ✅ Удаление реакции → `chat.reaction_removed`
6. ✅ Голосование в опросе → `poll.update`
7. ✅ Прочтение сообщений → `chat_marked_read` (синхронизация между вкладками)

## Рекомендации

### Опционально (низкий приоритет):

1. **Создание чата:** Можно добавить событие `chat.created` для мгновенного обновления списка чатов
2. **Закрытие голосования:** Можно добавить событие `poll.closed` с флагом `is_closed: true`
3. **Пересылка сообщений:** Уже отправляет `chat.message`, но можно добавить флаг `forwarded: true`

### Не требуется:

- **pin/notifications:** Локальные настройки пользователя, не критично для реального времени

## Выводы

Система имеет **полное покрытие всех критических операций** WebSocket событиями. Все действия, требующие мгновенной синхронизации между пользователями, корректно транслируются через WebSocket.

Незначительные упущения (создание чата, закрытие голосования) не влияют на функциональность мессенджера и могут быть добавлены по мере необходимости.
