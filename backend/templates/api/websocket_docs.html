WebSocket AsyncAPI

Отдельная документация realtime-контура CORP - HiRo. HTTP API продолжает жить в OpenAPI, а WebSocket-контур документируется отдельно через AsyncAPI, без смешивания двух протоколов в одной схеме.

Что описано

В схеме отражены клиентские action-сообщения и серверные realtime-события для чатов, уведомлений, typing, реакций, read-sync и procurement updates.

Почему отдельно

Swagger и ReDoc используют OpenAPI-схему HTTP API. Для WebSocket-протокола отдельный AsyncAPI-документ чище, понятнее и лучше масштабируется.
