# UX Улучшения: RecipientPicker в стиле Email-клиента

**Дата:** 2025-12-02  
**Коммит:** `e68404c`  
**Основано на:** Gmail, Outlook, Apple Mail

---

## 🎯 Проблема

Первоначальный дизайн RecipientPicker был:
- ❌ Неинтуитивным (выпадающие списки select)
- ❌ Требовал кликов для раскрытия секций (collapse)
- ❌ Не показывал контекст (кто уже выбран)
- ❌ Плохо работал на мобильных устройствах
- ❌ Не похож на привычные интерфейсы

---

## ✨ Решение: Email-style UI

### Вдохновение
Интерфейс создания заявления теперь похож на отправку email:

```
┌─────────────────────────────────────────────────┐
│ ✉️ Новое заявление                         [×] │
├─────────────────────────────────────────────────┤
│                                                 │
│ 🏢 Отделы:   [IT отдел] [HR] [+Выбрать отдел] │
│              ☑ Всем сотрудникам                │
│                                                 │
│ ✓ Кому:      [Иванов И.] [Петрова А.] _______ │
│              [+ Копия]                          │
│                                                 │
│ ─────────────────────────────────────────────── │
│                                                 │
│ Тип:         [📅 Отпуск ▼]                     │
│ Тема:        [_________________________]        │
│ ...                                             │
└─────────────────────────────────────────────────┘
```

## 🎨 Ключевые улучшения

### 1. Chip-based Input (Token Input)
**До:**
```html
<select multiple>
  <option>Иванов Иван</option>
  <option>Петрова Анна</option>
</select>
```

**После:**
```html
[Иванов И. ×] [Петрова А. ×] _начните вводить имя..._
                              └─ Иванов Игорь
                                 Иванова Мария
                                 Игнатьев Петр
```

**Преимущества:**
- ✅ Визуально понятно, кто выбран
- ✅ Один клик для удаления (×)
- ✅ Не занимает много места
- ✅ Как в Gmail/Slack/Telegram

---

### 2. Autocomplete с живым поиском

**Функционал:**
- Начинаете вводить имя → появляются подсказки
- Поиск по имени, фамилии, email, должности
- Показ аватаров (инициалы)
- Ограничение до 10 результатов
- Подсветка при hover

**Пример dropdown:**
```
┌───────────────────────────────────────┐
│ [ИИ] Иванов Иван                     │
│      Старший разработчик             │
├───────────────────────────────────────┤
│ [ИМ] Иванова Мария                   │
│      HR менеджер                     │
├───────────────────────────────────────┤
│ [ИП] Игнатьев Петр                   │
│      ivanov@company.com              │
└───────────────────────────────────────┘
```

---

### 3. To/CC разделение

**Паттерн из email:**
- **To (Кому):** основные получатели
- **CC (Копия):** наблюдатели
- Кнопка "+ Копия" для показа поля CC
- Визуальное отличие (цвет chips)

**Код:**
```javascript
// Основные получатели - синие chips
<span class="recipient-chip">Иванов И. ×</span>

// CC - серые chips
<span class="recipient-chip chip-cc">Петрова А. ×</span>
```

### 4. Компактная структура "рядами"

**Layout:**
```
Отделы:  [chips] [dropdown] [☑ Всем сотрудникам]
───────────────────────────────────────────────────
Кому:    [chips] _input с autocomplete_ [+ Копия]
───────────────────────────────────────────────────
Копия:   [chips] _input с autocomplete_
```

**CSS:**
```css
.recipient-row {
  display: flex;
  align-items: flex-start;
  gap: 0.75rem;
  border-bottom: 1px solid;

.recipient-label {
  min-width: 80px;  /* To:/CC:/Отделы: */

  flex: 1;  /* Растягивается */

```

## 📊 Сравнение: До vs После

| Критерий | До | После |
|----------|-----|--------|
| **Выбор пользователей** | Select dropdown | Autocomplete с чипами |
| **Видимость выбранных** | Скрыты в select | Chips всегда видны |
| **Поиск** | Нет | Живой поиск по имени |
| **Удаление** | Deselect в select | Клик на × в chip |
| **Мобильная адаптация** | Плохо | Адаптивный layout |
| **Визуальный контекст** | Нет | Аватары, должности |
| **Clutter** | Много текста | Компактно |
| **Скорость работы** | Медленно (скроллинг) | Быстро (поиск) |

## 🎯 UX метрики

### Количество кликов для добавления 3 получателей

**До:**
1. Клик "Настроить получателей"
2. Открыть select (клик)
3. Скроллинг к пользователю 1
4. Клик на пользователя 1
5. Скроллинг к пользователю 2
6. Клик на пользователя 2
7. Скроллинг к пользователю 3
8. Клик на пользователя 3

**Итого: 8 кликов + скроллинг**

**После:**
1. Ввод 2-3 символов имени
2. Клик на пользователя в dropdown
3. Повторить 2 раза

**Итого: 3 клика + минимальный ввод**

### Время выполнения задачи
- **До:** ~20-30 секунд (со скроллингом)
- **После:** ~5-10 секунд (с поиском)
