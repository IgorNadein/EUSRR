## Известные ограничения

1. **Legacy календари не редактируются**
   - "Компания", "Личный календарь", отделы - системные
   - Нельзя изменить их название или цвет
   - Это legacy поведение, сохранено для совместимости

2. **Подписка на календари**
   - Функция подписки работает только для новых Calendar записей
   - Legacy календари всегда видны пользователю

## Что делать, если не работает

### Событие не создается
- Проверьте консоль браузера (F12)
- Проверьте, что выбран календарь в селекторе
- Проверьте, что заполнены обязательные поля (название, дата)

### Селектор пустой
- Проверьте консоль: `window.calendarIntegration.calendars`
- Должен вернуть массив календарей
- Если пусто - проблема в calendarManager.js

### События дублируются
- Откройте DevTools → Network
- Проверьте параметры в GET запросах к `/api/v1/calendar/events/`
- Должны быть разные employee_id/department_id/calendar_id

### NaN ошибки
- Проверьте, что используется последняя версия кода (commit 6bf3cc4)
- Проверьте логи в консоли
- Убедитесь, что `meta[name="user-id"]` существует в HTML

## Тестовые данные

### Создание тестового календаря через Django shell

```bash
.venv/Scripts/python manage.py shell
```

```python
from calendar_app.models import Calendar
from employees.models import Employee

# Получаем текущего пользователя
employee = Employee.objects.get(id=1)

# Создаем тестовый календарь
calendar = Calendar.objects.create(
    name="Проект Альфа",
    description="Календарь проекта",
    owner=employee,
    color="#ff6b6b",
    is_public=False
)

print(f"Создан календарь: {calendar.id} - {calendar.name}")
```

### SQL запросы для проверки

```sql
-- Все календари пользователя
SELECT id, name, owner_id, color, is_public
FROM calendar_app_calendar
WHERE owner_id = 1;

-- События с calendar_id
SELECT id, title, calendar_id, employee_id, department_id
FROM calendar_app_event
WHERE calendar_id IS NOT NULL;

-- События в личном календаре (legacy)
SELECT id, title, employee_id
FROM calendar_app_event
WHERE employee_id = 1 AND calendar_id IS NULL;
```
