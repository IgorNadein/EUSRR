# Тестирование фильтров заявок

## ✅ Что было реализовано

### Frontend
- Добавлены 2 строки фильтров по датам с подписями:
  1. **"Дата создания заявления"** - фильтрует по дате создания заявки (поля "от" и "до" в одной строке)
  2. **"Период заявления"** - фильтрует по периоду самой заявки (поля "от" и "до" в другой строке)

### Backend API
- Добавлена поддержка 4 параметров фильтрации:
  - `created_from` - начало периода создания заявления  
  - `created_to` - конец периода создания заявления
  - `date_from` - начало периода заявления (пересечение)
  - `date_to` - конец периода заявления (пересечение)

## 🧪 Как протестировать

### Шаг 1: Откройте страницу заявок
```
http://localhost:3000/requests
```

### Шаг 2: Откройте панель фильтров
Нажмите на иконку фильтра (воронка) в правом верхнем углу

### Шаг 3: Протестируйте фильтры

#### Тест 1: Фильтр по дате создания
1. В блоке "Дата создания заявления"установите:
   - От: сегодняшняя дата
   - До: сегодняшняя дата
2. Должны отобразиться только заявки, созданные сегодня

#### Тест 2: Фильтр по периоду заявления
1. В блоке "Период заявления" установите:
   - От: сегодняшняя дата
   - До: дата через неделю
2. Должны отобразиться заявки, период которых пересекается с указанным диапазоном
   (например, отпуска, которые начинаются или заканчиваются в этот период)

#### Тест 3: Комбинированный фильтр
1. Установите оба фильтра одновременно
2. Должны отобразиться заявки, удовлетворяющие обоим условиям

### Шаг 4: Проверка работы
- Убедитесь, что счетчик активных фильтров обновляется правильно
- Нажмите "Очистить фильтры" - все фильтры должны сброситься
- Проверьте, что фильтры сохраняются при переключении страниц (пагинация)

## 📝 Примечания

- **Фильтр по дате создания** (`created_from`/`created_to`) - фильтрует по полю `created_at` заявки
- **Фильтр по периоду** (`date_from`/`date_to`) - фильтрует по полям `date_from` и `date_to` заявки
- Фильтры применяются как логическое И (AND) - заявка должна удовлетворять всем активным фильтрам
