## 📊 Исходная ситуация

### Проблемы архитектуры:

1. **Дублирование кода** - 3 разных способа загрузки сообщений
2. **Закомментированный код** - неиспользуемые модули в imports
3. **Несогласованность рендеринга** - разные API для создания сообщений
4. **Бесконечный лоадер** - в пустых чатах не скрывался loader

### Модули до рефакторинга:

- ❌ **`initialMessagesLoader.js`** - закомментирован, не используется (78 строк)
- ❌ **`chatWebSocket.js`** - дублирует userWebSocket.js (408 строк)
- ⚠️ **`createMessageElement()`** - создавал новый MessageRenderer каждый раз
- ⚠️ **`userWebSocket.js`** - баг с infinite loader в пустых чатах

## 🏗️ Финальная архитектура

### Поток загрузки сообщений:

```
1. MessageRenderer (создается первым)
   ↓
2. FormManager + MarkRead API
   ↓
3. Composer (отправка новых сообщений)
   ↓ использует messageRenderer
4. HistoryLoader (загрузка старых сообщений)
   ↓ использует messageRenderer
5. WebSocket (начальные сообщения)
   ↓ использует messageRenderer
6. Все компоненты используют ЕДИНЫЙ MessageRenderer
```

### Ответственность модулей:

| Модуль | Ответственность | Использует |
|--------|----------------|------------|
| `MessageRenderer` | Создание DOM элементов сообщений | - |
| `userWebSocket` | Initial messages через WS | MessageRenderer |
| `chatHistoryLoader` | Pagination (scroll up) | MessageRenderer |
| `chatComposer` | Pending messages при отправке | MessageRenderer |
| `chatDetail.js` | Координация инициализации | Все модули |
